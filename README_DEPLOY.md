# نشر نظام Mr. Delivery على الويب — دليل خطوة بخطوة

الهدف: موقع حقيقي برابط، بيانات مشتركة بين الجميع، وتسجيل دخول للموظفين والمناديب.
المكوّنات: **Supabase** (قاعدة البيانات + الدخول + التخزين) + **Cloudflare Pages** (استضافة مجانية).
الوقت المتوقع: 20–40 دقيقة.

---

## المرحلة ١: قاعدة البيانات (Supabase)

1. افتح https://supabase.com وسجّل حساب، ثم **New project**.
   - اسم المشروع: `mr-delivery`
   - اختر Region قريب (مثلاً Frankfurt / Bahrain إن توفّر).
   - حدّد Database Password واحفظه.
2. انتظر حتى يجهز المشروع (~دقيقتين).
3. من القائمة الجانبية افتح **SQL Editor → New query**، الصق **كامل** محتوى ملف `supabase_setup.sql`، واضغط **Run**.
   - هذا يبني الجداول، الصلاحيات، والدوال، ويضيف مناديب تجريبية.

### إنشاء حسابات الموظفين
4. افتح **Authentication → Users → Add user** وأنشئ هذه الحسابات (اختر كلمة مرور لكل واحد، وفعّل **Auto Confirm User / Email confirmed**):

| البريد (Email) | الدور في النظام |
|---|---|
| `admin@mrd.app` | مدير النظام (يشوف كل الشركات) |
| `ops@mrd.app` | مدير العمليات (كل الشركات) |
| `finance@mrd.app` | المالية |
| `talabat@mrd.app` | مشرف الطلبات (Talabat فقط) |
| `snoonu@mrd.app` | مشرف سنونو (Snoonu فقط) |
| `aramex@mrd.app` | مشرف أرامكس (Aramex فقط) |

> الأدوار مربوطة بالبريد داخل الكود. تقدر تغيّر العناوين لاحقاً من ملف `src/App.jsx` (قسم `STAFF_BY_EMAIL`).

5. **مهم للأمان:** افتح **Authentication → Sign In / Providers** (أو Settings) و**عطّل التسجيل الذاتي** (Disable "Allow new users to sign up") حتى لا يقدر أحد ينشئ حساب موظف بنفسه.

### نسخ مفاتيح المشروع
6. افتح **Project Settings → API** وانسخ:
   - **Project URL** (مثل `https://xxxx.supabase.co`)
   - **anon public key**

## المرحلة ٢: رفع الكود ونشره (Cloudflare Pages)

أسهل طريقة بدون تثبيت أي برامج على جهازك:

1. أنشئ حساب على https://github.com ثم **New repository** باسم `mrdelivery-web` (Private).
2. ارفع محتويات مجلد المشروع (كل الملفات: `index.html`, `package.json`, `vite.config.js`, `src/`, إلخ) للمستودع — تقدر تسحبها وتفلتها في صفحة GitHub (Add file → Upload files).
3. افتح https://dash.cloudflare.com → **Workers & Pages → Create → Pages → Connect to Git**، واختر مستودع `mrdelivery-web`.
4. في إعدادات البناء:
   - **Framework preset:** Vite
   - **Build command:** `npm run build`
   - **Build output directory:** `dist`
5. أضف **Environment variables** (متغيرات البيئة):
   - `VITE_SUPABASE_URL` = رابط مشروعك من الخطوة ٦
   - `VITE_SUPABASE_ANON_KEY` = مفتاح anon من الخطوة ٦
6. اضغط **Save and Deploy**. بعد دقيقتين تحصل على رابط مثل `https://mrdelivery-web.pages.dev`.

> بديل للاستضافة: **Netlify** (نفس الفكرة) — Build command `npm run build`، Publish directory `dist`، ونفس متغيري البيئة.

## المرحلة ٣: الاستخدام

- **الموظفون:** افتح الرابط → تبويب "دخول الموظفين" → البريد + كلمة المرور.
- **المناديب:** نفس الرابط → تبويب "دخول المناديب" → رقم الهاتف + كلمة المرور (الافتراضية `1234`).
- سجّل دخول كـ `admin@mrd.app`، احذف المناديب التجريبيين، وأضف مناديبك الحقيقيين (زر **تسجيل جماعي**).
- وزّع نفس الرابط على الجميع — الكل يشوف نفس البيانات فوراً.

## ملاحظات مهمة

- **التكلفة:** تبدأ مجاناً. الخطة المجانية في Supabase **توقف المشروع بعد أسبوع خمول** — طالما تُستخدم يومياً ما بتتوقف. للتشغيل الجاد فعّل خطة Supabase Pro (25$ شهرياً) لإزالة هذا القيد. الاستضافة على Cloudflare مجانية.
- **الصلاحيات:** الموظفون يدخلون بحسابات Supabase حقيقية. المناديب يدخلون عبر دوال آمنة تُرجّع/تعدّل **بيانات المندوب نفسه فقط** (لا يرى بيانات غيره).
- **قيد الإصدار الأول:** بيانات الموظفين تُحفظ ككتلة واحدة (آخر حفظ يغلب). لفريق عمليات صغير هذا مقبول. لاحقاً نقدر نطوّرها لجداول منفصلة إذا احتجنا تعديلات متزامنة كثيفة.
- **الإيصالات:** تُحفظ مصغّرة داخل التحويل. لو زاد الحجم لاحقاً ننقلها لتخزين Supabase Storage.
- **تحديث تطبيق المندوب:** يتحدّث عند الفتح وكل ٣٠ ثانية تلقائياً (وبعد أي إجراء).
